1. 뒤로가기 버튼
- 클릭 시 이전 화면으로 이동한다.
-------------------------------------------------------------------------------
2. 직접 루틴 만들기 탭 버튼
- 클릭 시 3번 화면으로 전환한다.
- 현재 화면(추천 탭)에서 입력·선택한 내용은 저장되지 않고 초기화된다.
-------------------------------------------------------------------------------
3. 피부 고민 맞춤 추천 탭 버튼 (현재 선택 상태)
- 이미 선택된 상태이므로 클릭해도 별도 동작 없이 선택됨 색상을 유지한다.
-------------------------------------------------------------------------------
4. "이렇게 추천했어요" 표시 영역 (피부 타입 / 피부 고민 / 피부 민감도)
- 사용자가 피부 프로필에 등록한 값을 그대로 불러와 표시하며, 조회 전용으로 클릭 이벤트는 없다.
- 피부 프로필 미등록 상태로는 이 화면에 진입할 수 없다(홈 화면 분기 로직상 프로필 등록이 선행 조건).
-------------------------------------------------------------------------------
5. 추천 루틴 미리보기 영역
- 사용자의 피부 타입·고민·피부 민감도 조합을 AI(Claude API)에 전달하여 맞춤 루틴을 생성한다(실시간 생성 방식). 
- 생성된 루틴은 단계(예: 클렌징 → 진정 토너 → 시카 세럼 → 수분 크림)와 단계별 추천 제품 카테고리로 구성된다. 
- 제품 카드에는 제품명과 제품 사진만 표시하며, 가격·재고·판매처 등 실시간 정보는 노출하지 않는다. 
- AI 응답 생성 중에는 로딩 상태를 노출한다. 
- AI 서버 응답 실패/타임아웃 시 - "추천 루틴을 불러오지 못했어요. 다시 시도해주세요" 안내와 재시도 버튼을 노출한다. 
- 제품 카드는 클릭 이벤트가 없는 정보 표시용 요소이다.
-------------------------------------------------------------------------------
6. 추천 루틴 적용 버튼
- 클릭 시 현재 노출된 AI 추천 루틴(단계+제품 구성)을 사용자의 루틴으로 선택 확정한다(아직 최종 등록 아님). 
- 적용 완료 시 버튼 상태가 변경(예: "적용됨" 또는 체크 아이콘)되며, 하단 "루틴 설정 완료" 버튼이 활성화된다. 
- 이미 적용된 상태에서 재클릭하면 별도 동작은 없다.
-------------------------------------------------------------------------------
7. 다른 루틴 추천받기 버튼
- 클릭 시 같은 프로필 조건(피부 타입·고민·피부 민감도)에 매칭되는 사전 등록 루틴 목록 중 다른 루틴으로 교체하여 "추천 루틴 미리보기 영역"의 내용을 갱신한다.
- 짧은 시간 내 반복 클릭 시(예: 5초 이내) 버튼을 비활성화 처리한다(API 과다 호출 방지).
- "추천 루틴 적용" 버튼을 누른 이후 이 버튼을 클릭하면 기존 적용 상태를 초기화하고 새 루틴으로 대체한다.
-------------------------------------------------------------------------------

8. 루틴 설정 완료 버튼
- "추천 루틴 적용"을 아직 클릭하지 않은 상태에서는 버튼이 비활성화되어 클릭할 수 없다.
- "추천 루틴 적용"을 완료한 상태에서는 버튼이 활성화되며, 클릭 시 루틴 등록을 처리하고 4번  화면으로 이동한다(기록 시작 처리).
-------------------------------------------------------------------------------
